---
name: Just do it — don't ask first
description: User wants Claude to act immediately without confirming intent
type: feedback
---

Don't ask clarifying questions before taking obvious actions. If the intent is clear (e.g., "write this component", "add this to CSS"), just do it.

**Why:** User finds the back-and-forth slow and frustrating. They expect immediate execution.

**How to apply:** When given code to write or a clear instruction, execute first. Only ask if something is genuinely ambiguous and the wrong choice would cause real damage.
